HTML

  • Hypertext Markup Language (HTML) is used to create the structure of a webpage
  • If a webpage is a house, HTML is the foundation and frame
  • The head element contains information, or metadata, that helps the browser render the webpage correctly. This information is not visible to the user
  • The body element defines the body of the document. These elements are what are visible to the user
  • When using HTML code, it is in best practice to use semantic HTML elements, or elements whose name describe the content and role it plays, as much as possible.
  • While element and tag tend to get used interchangably, an element is the information held between the opening tag through the closing tag. Whereas a tag is only what is contained within the angle brackets.

CSS

  • Cascading Style Sheets (CSS) is a programming language used to define how webpage content is presented to users via colors, aesthetics, visual layout, and organization
  • If the webpage is a house, CSS is the paint and style of it.
  • There are three main methods of using CSS;
    • Inline CSS: styling an element directly in the HTML file. Quick but inflexible.
    • Internal CSS Style Sheet: contains CSS rules in the head sections of the HTML file that only apply to the active page.
    • External CSS Style Sheet: the CSS rules are written in a separate file and linked to the HTML document using a link element. This makes changing the design easier.
  • CSS Rules are made up of a selector and declarations. The declarations are made up of properties and values.
  • A Selector defines the attributes or elements to which the declarations apply.
  • Declarations contain the property and value we want to apply
  • In CSS, it is believed that every HTML element is essentially a box that consists of margins outside of the element, and the element's border, padding, and content.
  • It is best to practice DRY (do not repeat yourself) coding by combining two or more elements that share the same styling using a comma.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project
  • Note that git commands are different than bash commands, and each command must start with "git"
  • checkout moves from the working branch into a different branch
  • -b creates a new branch
  • So the code git checkout -b branch name will create a new branch and move you from where you are into it
  • git status verifies the branch that we are currently on
  • add adds modifications in the current working branch to the staging area
  • -Aindicates we want to add all changes
  • So the code git add -A will add all the modifications from the working branch to the staging area
  • push will push the changes we have locally to our remote GitHub branch

JavaScript

  • JavaScript is what makes a website interactive
  • JavaScript is powerful, versatile, and relatively user-friendly
  • It uses Control Flow to operate, which is the order in which the computer executes a code in a JavaScript file. JavaScript runs from top down.
  • Developers can use Conditional Statements and loops to interrupt the control flow
  • JavaScript refers to variables as containers because they hold related data in the code
  • Truthy: when the computer checks a value against an "if" statement and finds that the conditions are met and will carry out the conditional statement
  • Falsy: when the computer checks a value against an "if" statment and finds that the conditions are not met, and therefore will ignore the conditional statment
